# Setup

## Requirements

- Node.js 18 or newer.
- A Discord application with a bot user.
- A Discord bot token.
- Discord OAuth client ID and client secret for dashboard sign-in.
- A Supabase Postgres database URL.
- A Discord server where the bot has the required permissions.

## Install dependencies

Install each app separately:

```bash
npm install
npm --prefix dashboard install
npm --prefix landing install
npm --prefix documentation install
```

## Local or VPS setup

Use this path when the dashboard can save `.setup-state.json` on disk.

1. Create `.env` at the repository root.
2. Set the dashboard login values:

```env
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=replace_with_a_long_random_string
```

3. Start the dashboard:

```bash
npm --prefix dashboard run dev
```

4. Open `/setup`.
5. Complete the setup wizard with Discord OAuth, bot token, guild, channel, owner, and database details.
6. Start the bot after setup completes:

```bash
npm start
```

The bot startup path can hydrate `DISCORD_TOKEN`, `DATABASE_URL`, and owner data from `.setup-state.json` before environment validation runs.

## Cloud setup

Use this path for Vercel or serverless hosts where local files should not be treated as durable storage.

```env
SETUP_COMPLETE=true
OWNER_DISCORD_ID=your_discord_user_id
SELECTED_GUILD_ID=your_discord_server_id
LOG_CHANNEL_ID=your_log_text_channel_id
DISCORD_TOKEN=your_discord_bot_token
DISCORD_CLIENT_ID=your_discord_application_client_id
DISCORD_CLIENT_SECRET=your_discord_application_client_secret
DATABASE_URL=postgresql://postgres.project-ref:password@aws-region.pooler.supabase.com:6543/postgres?sslmode=require
NEXTAUTH_URL=https://your-dashboard-domain.example
NEXTAUTH_SECRET=replace_with_a_long_random_string
```

Optional values:

```env
LFG_CHANNEL_ID=your_lfg_text_channel_id
DISCORD_BOT_TOKEN=your_discord_bot_token
BOT_POSTGRES_POOL_MAX=2
POSTGRES_POOL_MAX=1
DEBUG=true
PG_SSL_MODE=require
```

For Supabase, use the Transaction Pooler URL on port `6543` with `sslmode=require`. Keep pool sizes low when the bot and dashboard share the same database.

## Discord permissions

Enable these bot intents in the Discord Developer Portal:

- `Guilds`
- `GuildVoiceStates`
- `GuildMessages`

Give the bot these permissions in the server:

- `View Channel`
- `Send Messages`
- `Manage Channels`
- `Move Members`
- `Moderate Members`
- `Ban Members`
- `Manage Roles`, if auto-role features are used.
